Codetrack

1. Marcações

    Sistema provê marcações que auxiliam na rastreabilidade de código, mas também permite que se utilizem marcações próprias.
    As marcações indicam no código quais os requisitos estão sendo atentidos por quais artefatos de código. Também podem indicar 
    quais correções que o código alterado atende.
    
    1.1 - Marcação de código Java
    
    As marcações em código Java são feitas através de anotações que permitem indicar quais requisitos o artefato atende. São elas:
    
        A. Solution
        
    Esta anotação se refere a especificação de solução geral que será atendida pelo código.
    
        @Solution( id='SL1', 
                   name='Solution SL1',
                   document='SL1-espec.docx', 
                   authors = { 
                                @Author( email='eydhhd@kdkjskjd.com', 
                                         name='João Almeida'),
                                @Author( email='yujjh@kskjksjd.com',
                                         name='Paulo J')
                             }
                   iterations = { 
                                    @Iteration( id='IT1', beginAt='2015-04-10', endAt='2014-04-22')
                                }
                 )
         
        
        B. Requirement
        C. Rule
        D. Fix
        E. Author
        F. Release
